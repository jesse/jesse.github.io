Launching the blog

I wanted a place for longer notes that can live next to the project pages already on this domain, so this blog now shares the same simple GitHub Pages setup.

The design goal was straightforward: keep prose narrow, make code pleasant to read, and stay out of the way when the writing is doing the real work.

Inline code like make build should feel distinct without pulling attention away from the paragraph around it.
